APLLO SQL の意味論

APLLO SQLの意味論の記述#

APLLO SQL は、次のように細分化される:

  • APLLO SQL
    • APLLO DDL
      • APLLO CREATE TABLE: TcreateTableCommandT_{createTableCommand}TcreateTableCommand​
      • APLLO ALTER TABLE: TalterTableCommandT_{alterTableCommand}TalterTableCommand​
      • APLLO DROP TABLE: TdropTableCommandT_{dropTableCommand}TdropTableCommand​
    • APLLO DML
      • APLLO SELECT: TselectCommandT_{selectCommand}TselectCommand​
      • APLLO INSERT: TinsertCommandT_{insertCommand}TinsertCommand​
      • APLLO UPDATE: TupdateCommandT_{updateCommand}TupdateCommand​
      • APLLO DELETE: TdeleteCommandT_{deleteCommand}TdeleteCommand​

APLLO DROP TABLE (IF EXISTS)#

文法#

DROP TABLE IF EXISTS {T_tableName};
T_tableName :=  文字列;

意味論#

if {T_tableName} in {Catalog}:  for all v_i in {TableCatalog}:
DEACTIVATE "{T_tableName}.{v_i}";

else:

;

APLLO ALTER TABLE (ADD, w/o NOT NULL)#

文法#

ALTER TABLE {T_tableName} ADD COLUMN {T_colDefWithoutNotNull};
T_tableName :=  文字列;
T_colDefWithoutNotNull :=  T_colName  T_colType  T_colConstraintWithoutNotNull*;
T_colConstraintWithoutNotNull :=  PRIMARY KEY;
T_colType :=  INTEGER  | TEXT;

互換 / 非互換#

互換ALTER

意味論#

if {T_tableName} in {Catalog} && {T_colName} in {TableCatalog}:

E_duplicateColumn;

else if {T_tableName} in {Catalog}:

CREATE TABLE {v_(current+1)} (  {{T_colList} in {VersionCatalog_{v_current}}},  {T_colDefWithoutNotNull});
-- 自動アップグレードINSERT INTO {T_tableName}.{v_(current+1)} {T_colList in {VersionCatalog_{v_current}}}  SELECT {T_colList in {VersionCatalog_{v_current}}} FROM {T_tableName}.{v_current};
DEACTIVATE {T_tableName}.{v_current};

else:

E_undefinedTable;

APLLO ALTER TABLE (ADD, w/ NOT NULL)#

文法#

ALTER TABLE {T_tableName} ADD COLUMN {T_colDefWithNotNull};
T_tableName :=  文字列;
T_colDefWithNotNull :=  T_colName  T_colType  T_colConstraintWithNotNull*;
T_colConstraintWithNotNull :=  NOT NULL  T_colConstraintWithoutNotNull*;
T_colConstraintWithoutNotNull :=  PRIMARY KEY;
T_colType :=  INTEGER  | TEXT;

互換 / 非互換#

非互換ALTER

ALTER後にエラーになる Raw SELECT, Raw INSERT の例:

ALTER TABLE {T_tableName} ADD COLUMN x INTEGER NOT NULL;INSERT INTO {T_tableName} (... (xは含まない)) VALUES (...);

意味論#

if {T_tableName} in {Catalog} && {T_colName} in {TableCatalog}:

E_duplicate_column;

else if T_tableName in Catalog:

CREATE TABLE {v_(current+1)} (  {{T_colList} in {VersionCatalog_{v_current}}},  {T_colDefWithNotNull});

else:

E_undefined_table;

APLLO ALTER TABLE (DROP)#

文法#

ALTER TABLE {T_tableName} DROP COLUMN {T_colName};
T_tableName :=  文字列;

互換 / 非互換#

非互換ALTER

ALTER後にエラーになる Raw SELECT, Raw INSERT の例:

ALTER TABLE T_tableName DROP COLUMN x;SELECT x FROM T_tableName;

意味論#

if {T_tableName} in {Catalog} && {T_colName} not in {TableCatalog}:

  E_undefined_column;

else if {T_tableName} in {Catalog}:

CREATE TABLE {v_(current+1)} (  { {{T_colList} in {VersionCatalog_{v_current}}} - {T_colName}});

else:

E_undefined_table;
